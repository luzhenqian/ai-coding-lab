# Implementation Plan: Database Schema Design & Initialization

**Branch**: `001-db-schema-init` | **Date**: 2026-03-12 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/001-db-schema-init/spec.md`

## Summary

Define the PostgreSQL + pgvector database schema for the AskBook HR
RAG Chatbot using Drizzle ORM. Four tables (documents, chunks,
conversations, messages) with typed schemas, an HNSW vector index on
the embedding column, cascade delete constraints, and a Docker Compose
setup for local development. Drizzle Kit manages migrations.

## Technical Context

**Language/Version**: TypeScript (strict mode) on Node.js 20+
**Primary Dependencies**: Drizzle ORM, Drizzle Kit, drizzle-orm/pg-core, pgvector (drizzle-orm/pg-core vector type)
**Storage**: PostgreSQL 16+ with pgvector 0.7+ extension
**Testing**: Vitest (unit/integration), direct DB assertions via Drizzle
**Target Platform**: Next.js 16+ App Router (local dev via Docker Compose)
**Project Type**: Web application (Next.js full-stack)
**Performance Goals**: Cosine similarity top-5 query < 100ms on 10k chunks
**Constraints**: 1536-dimensional embeddings (text-embedding-3-small), HNSW index required
**Scale/Scope**: Single-user local dev; schema supports future multi-doc, multi-conversation use
