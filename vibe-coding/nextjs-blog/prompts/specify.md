构建一个功能完整的个人博客系统，具体要求如下：

## 用户系统
- 支持 GitHub OAuth 登录和邮箱密码注册/登录
- 用户有三种角色：管理员（admin）、作者（author）、读者（reader）
- 管理员可以管理所有内容和用户
- 作者可以创建、编辑、删除自己的文章
- 读者只能浏览和评论

## 文章系统
- 文章支持 Markdown 编写，实时预览
- 文章有标题、内容、摘要、封面图、标签、分类
- 文章有三种状态：草稿（draft）、已发布（published）、已归档（archived）
- 支持按标签、分类、关键词搜索和筛选文章
- 文章详情页展示阅读量统计
- 支持文章的 SEO 元信息配置（title、description、og:image）

## 标签与分类
- 标签（Tag）是多对多关系，一篇文章可以有多个标签
- 分类（Category）是单选关系，一篇文章属于一个分类
- 支持分类的增删改查

## 前台页面
- 首页：展示最新文章列表，支持分页
- 文章详情页：渲染 Markdown 内容，显示作者信息、发布时间、标签
- 标签页：按标签筛选文章
- 分类页：按分类筛选文章
- 关于页：静态页面

## 后台管理
- Dashboard：展示文章数量、总阅读量等统计数据
- 文章管理：文章列表、新建、编辑、删除、状态切换
- 分类管理：增删改查
- 标签管理：增删改查（标签使用时不可删除）

## 非功能需求
- 响应式设计，适配移动端和桌面端
- 使用 ISR（Incremental Static Regeneration）优化首页和文章页性能
- 图片上传支持（可先使用本地存储，预留云存储接口）
