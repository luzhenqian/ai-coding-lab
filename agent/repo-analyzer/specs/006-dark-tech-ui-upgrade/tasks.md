## Dependencies & Execution Order

### Phase Dependencies

- **Setup (Phase 1)**: No dependencies — can start immediately
- **Foundational (Phase 2)**: Depends on Setup completion — BLOCKS all user stories
- **US1 Dark Tech Visual (Phase 3)**: Depends on Foundational — MVP target
- **US2 Theme Switching (Phase 4)**: Depends on US1 (needs components migrated to CSS variables first)
- **US4 Component Consistency (Phase 5)**: Depends on US1 (needs components using shadcn/ui)
- **US3 Animations (Phase 6)**: Depends on US1 (needs final component structure to animate)
- **Polish (Phase 7)**: Depends on all user stories complete

### User Story Dependencies

- **US1 (P1)**: Can start after Foundational — the core visual redesign
- **US2 (P2)**: Depends on US1 — theme switching requires components already using CSS variables
- **US4 (P2)**: Depends on US1 — consistency audit requires components already using shadcn/ui
- **US3 (P3)**: Depends on US1 — animations wrap around final component structure
- **US2 and US4** can run in parallel after US1

### Within Each User Story

- Components that touch different files can be parallelized ([P] marked)
- Layout/page changes should come before component changes
- Shared utilities before component-specific code
